üìã Overview

My Agent is a CLI-based AI assistant that can read/write files, execute commands, browse the web, and control your desktop - all with persistent memory that learns from every conversation.

üß†

Persistent Memory

Conversations are saved automatically with semantic search. The agent remembers context from past sessions.

üîß

Tool Execution

Read/write files, run shell commands, fetch URLs, and control the desktop with mouse/keyboard.

ü§ñ

Multi-Agent

Spawn specialized agents for complex tasks - coding, research, analysis, and more.

üé®

Dynamic Skills

Create and execute custom skills using Rhai scripting with LLM-generated code.

üöÄ Quick Start

my-agent -P Start interactive mode with persistent memory
my-agent -P -r Resume last conversation
my-agent doctor Run diagnostics and self-healing

üí¨ interactive

Start an interactive chat session with tool capabilities.

my-agent interactive [OPTIONS]
Option Description
-P, --persistent Enable persistent memory (save conversations)
-r, --resume Resume most recent conversation

üß† memory

Manage the memory database (embeddings, stats, cleanup).

my-agent memory stats Show database statistics
my-agent memory search "query" Semantic search across conversations
my-agent memory add --content "..." --source user Add knowledge to the database
my-agent memory cleanup --keep 100 Remove old conversations
my-agent memory init-embeddings Initialize embedding model

‚öôÔ∏è config

Configure the agent settings.

my-agent config --set-api-key KEY Set OpenRouter API key (stored in keyring)
my-agent config --set-model ROLE MODEL Set model for a role (orchestrator, code, research, etc.)

üè• doctor

Run diagnostics and self-healing.

my-agent doctor --update Self-update from GitHub releases

Checks: config, API keys, system requirements, network, filesystem, CVEs, outdated dependencies.

üß† Memory System

Persistent memory with semantic search powered by SQLite + vector embeddings.

Database: ~/.local/share/my-agent/memory.db

Features

  • Full-text search (FTS5) with Porter stemming
  • Vector embeddings (OpenAI text-embedding-3-small, 1536 dims)
  • Hybrid search combining keyword + semantic
  • Knowledge base for storing facts
  • LRU cache for embedding lookups

Memory Tables

TableDescription
conversationsConversation records with embeddings
message_embeddingsPer-message embeddings for fine-grained search
knowledgeUser facts and preferences
conversations_ftsFTS5 index for conversations
messages_ftsFTS5 index for messages
